In the genome browser

react-msaview ships as a JBrowse 2 plugin (jbrowse-plugin-msaview), so the same viewer — alignment, phylogenetic tree, and all — runs as a view inside JBrowse, stacked alongside a Linear Genome View with gene annotation and multiple-alignment tracks. Every view here is opened by a declarative link: the whole state lives in the URL, with no saved session to set up.

A combined view: factor XII lost in whales

The link below opens one JBrowse view with two panels. On top, a Linear Genome View of the human F12 locus (coagulation factor XII) showing the NCBI RefSeq gene track and a UCSC Multiz mammalian DNA alignment. Below, the react-msaview panel: the F12 coding alignment across mammals with its inferred tree.

F12 is intact across land mammals and in the semi-aquatic hippopotamus, but is disabled by frameshifts and premature stop codons in cetaceans — whales, dolphins and porpoises — one of many genes lost in the transition to fully aquatic life. The tree groups the cetaceans together (with hippo as their closest relative), and a single shared frameshift is visible in the alignment. It is exactly the cross-species, nucleotide-level evidence an MSA editor would let you curate and lift over to many species.

How the links are built

The standalone links carry a react-msaview model snapshot in ?data= — file locations for the alignment (msaFilehandle), tree (treeFilehandle) and domains (gffFilehandle), plus display options like colorSchemeName. The combined link uses a JBrowse session spec: ?config=…&session=spec-{"views":[…]}, where each entry launches a view — a LinearGenomeView with its assembly, location and tracks, and an MsaView pointed at an alignment file. A combined view loaded from a cross-origin config will ask you to trust the plugin once before it renders.
