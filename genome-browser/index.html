In the genome browser

react-msaview runs as aJBrowse 2 plugin, so the alignment and tree open as a view inside JBrowse — connected to a Linear Genome View and, withjbrowse-plugin-protein3d, to the protein's 3D structure. Each example is one declarative link: the whole multi-view session lives in the URL.

Genome ↔ alignment ↔ 3D structure

One link opens three connected views sharing a coordinate system — the human TP53gene (RefSeq + ClinVar), the p53 ortholog alignment, and the AlphaFold structure — and lights one annotated feature across all three at once. Here the p53nuclear export signal (residues 339–350) is magenta on the structure, a band over the gene, and the matching alignment columns; click any residue, motif, or domain and the others follow.

Protein ↔ genome, on other genes

The same connected alignment-to-genome link without the structure view:

  • SRC ↗ — the Src-family kinase alignment beside the SRC gene.
  • BRAF V600E ↗ — the RAF family on the invariant V600 codon, with a ClinVar pathogenic-variant track.
  • TP53 R248 ↗ — p53 across vertebrates on the R248 hotspot, with ClinVar.

How the links are built

Each link is a JBrowsesession spec. A pinnedLinearGenomeView id is shared by the MsaView andProteinView through connectedViewId; transcript models, alignments, and ClinVar data are regenerated from public sources by the scripts under scripts/.
